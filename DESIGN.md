# Somnus CBT-I: 数据驱动的睡眠认知行为疗法系统设计文档

## 项目愿景
构建一个基于数据驱动的、个性化的睡眠认知行为疗法(CBT-I)系统，通过多维度数据分析自动生成治疗任务，并根据患者反馈动态调整治疗方案。

## 模块化设计

### 1. 数据输入模块
#### 1.1 心理测评量表
- **DBAS-30** (已实现)：评估睡眠相关认知失调
- **PSQI** (需新增)：匹兹堡睡眠质量指数，评估过去一个月睡眠质量
- **ISI** (可选)：失眠严重程度指数
- **GAD-7/PHQ-9** (可选)：焦虑抑郁筛查

#### 1.2 睡眠日志
- 基础睡眠参数：上床时间、入睡时间、醒来时间、起床时间
- 睡眠质量主观评分：1-5分
- 白天困倦程度
- 夜间觉醒次数及时长

#### 1.3 生理数据集成
- **Apple Health/Google Fit API**：心率、HRV、血氧、体动数据
- **可穿戴设备SDK**：Fitbit、Garmin、Oura Ring、Withings
- **睡眠阶段分析**：深睡、浅睡、REM睡眠、清醒时长
- **环境数据**：卧室温度、光照、噪音水平（通过手机传感器）

### 2. 数据处理与分析引擎

#### 2.1 数据标准化
```typescript
interface StandardizedSleepData {
  date: string;
  metrics: {
    efficiency: number;           // 睡眠效率
    latency: number;             // 入睡潜伏期(分钟)
    totalSleep: number;          // 总睡眠时长(分钟)
    waso: number;               // 觉醒后清醒时间
    awakenings: number;          // 夜间觉醒次数
    remPercentage: number;       // REM睡眠占比
    deepSleepPercentage: number; // 深睡占比
    subjectiveQuality: number;   // 主观睡眠质量(1-5)
  };
  cognitiveScores: {
    dbas: DBASScore;
    psqi: PSQIScore;
  };
  physiologicalData: {
    heartRateVariability: number;
    restingHeartRate: number;
    movementCount: number;
  };
}
```

#### 2.2 分析维度
- **睡眠效率趋势分析**：7天、30天移动平均
- **认知失调识别**：DBAS各维度异常值检测
- **行为模式分析**：就寝时间规律性、周末补偿行为
- **生理指标关联分析**：HRV与睡眠质量相关性
- **治疗响应预测**：基于历史数据的疗效预测模型

### 3. CBT任务生成系统

#### 3.1 任务类型体系
```typescript
type CBTTaskType = 
  | 'cognitive_restructuring'    // 认知重建
  | 'sleep_restriction'         // 睡眠限制
  | 'stimulus_control'          // 刺激控制
  | 'relaxation_training'       // 放松训练
  | 'sleep_hygiene'             // 睡眠卫生
  | 'mindfulness'               // 正念冥想
  | 'paradoxical_intention';    // 矛盾意向法
```

#### 3.2 个性化任务生成算法
```
输入: 用户数据 + 治疗阶段 + 历史任务表现
      ↓
1. 识别主要问题领域
   - 认知失调(DBAS高分) → 认知重建任务
   - 睡眠效率低 → 睡眠限制/刺激控制
   - 焦虑情绪高 → 放松训练/正念
      ↓
2. 匹配任务库
   - 难度分级(初级/中级/高级)
   - 时间要求(5min/15min/30min)
   - 执行场景(日间/睡前/夜间)
      ↓
3. 生成个性化任务描述
   - 基于具体数据点定制说明
   - 包含成功案例参考
      ↓
4. 设置任务参数
   - 执行时间窗
   - 预期效果指标
   - 替代方案准备
```

#### 3.3 任务库示例
```yaml
cognitive_restructuring:
  - id: "cr-001"
    title: "挑战不合理睡眠信念"
    difficulty: "beginner"
    duration: 15
    description: "针对'如果今晚睡不好，明天就完了'信念，列出3个替代想法"
    trigger_condition: "dbas.consequences > 4.0"
    
sleep_restriction:
  - id: "sr-001"
    title: "睡眠时间窗优化"
    difficulty: "intermediate"
    duration: 0
    description: "将卧床时间限制在X小时，即使感到困倦也不提前上床"
    trigger_condition: "efficiency < 85% and latency > 30"
```

### 4. 动态调整与推荐系统

#### 4.1 反馈收集机制
- 任务完成度评分(1-5星)
- 执行难度反馈(太易/适中/太难)
- 主观效果评价(无效/一般/有效/非常有效)
- 副作用报告(日间困倦、焦虑加重等)

#### 4.2 调整策略
```typescript
interface AdjustmentStrategy {
  // 成功完成 → 渐进增加难度
  if (completionRate > 80% && effectiveness > 3) {
    increaseTaskDifficulty();
    extendSleepWindow(); // 睡眠限制中逐步延长卧床时间
  }
  
  // 执行困难 → 提供替代方案或降低难度
  if (completionRate < 50% || difficultyFeedback === 'too_hard') {
    provideAlternativeTask();
    reduceTaskComplexity();
  }
  
  // 效果不佳 → 切换干预方式
  if (effectiveness < 2.5) {
    switchInterventionType();
    addSupportiveComponents();
  }
}
```

#### 4.3 治疗阶段管理
- **第1周(评估期)**：基线数据收集，轻度干预
- **第2-4周(强化期)**：核心CBT-I技术实施
- **第5-8周(巩固期)**：技能强化，预防复发
- **第9-12周(维持期)**：定期随访，长期维持

## 创新亮点

1. **数据驱动个性化**：基于多维度数据的精准任务生成
2. **动态适应算法**：根据反馈实时调整治疗策略
3. **综合数据集成**：主观测评 + 客观生理数据 + 行为日志
4. **科学可视化**：将复杂睡眠数据转化为直观见解
5. **模块化架构**：支持灵活扩展和定制化部署
6. **循证医学基础**：所有干预措施基于CBT-I临床指南

## 技术挑战与解决方案

### 挑战1: 数据隐私与安全
- **解决方案**：本地数据存储优先，端到端加密，匿名化数据分析

### 挑战2: 算法可解释性
- **解决方案**：透明决策日志，提供算法决策依据说明

### 挑战3: 用户依从性
- **解决方案**：游戏化设计，渐进式任务，个性化提醒

### 挑战4: 临床有效性验证
- **解决方案**：与医疗机构合作试点，收集疗效数据

## 成功指标

1. **用户体验**：任务完成率 > 70%，用户满意度 > 4/5
2. **治疗效果**：睡眠效率提升 > 15%，DBAS分数降低 > 20%
3. **技术性能**：页面加载时间 < 2s，API响应时间 < 200ms
4. **系统可用性**：系统正常运行时间 > 99.5%
